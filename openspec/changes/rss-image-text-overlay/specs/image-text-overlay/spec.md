## ADDED Requirements

### Requirement: 在图片底部叠加标题和发布时间
处理完成的 320×240 JPEG 图片 SHALL 在底部渲染一个半透明黑色横条，横条内包含白色的条目标题（一行或多行）和发布时间（固定末行）。

#### Scenario: 标题单行可容纳
- **WHEN** 条目标题渲染宽度不超过可用宽度（图片宽度减去左右各 6px 边距）
- **THEN** 标题以单行渲染，横条高度为 28px（时间行 14px + 标题行 14px）

#### Scenario: 标题需要换行
- **WHEN** 标题单行超出可用宽度，且折行后总行数不超过最大行数（3 行）
- **THEN** 在单词/字符边界处折行，每行单独渲染，横条高度随行数增加（每增加一行加 14px）

#### Scenario: 标题超出最大行数后截断
- **WHEN** 标题折行后仍超过最大行数（3 行）
- **THEN** 保留前 3 行内容，最后一行末尾替换为省略号（…），省略号后整体宽度不超出可用宽度

#### Scenario: 发布时间格式
- **WHEN** 条目有 PublishedAt 时间
- **THEN** 时间以 `2006-01-02 15:04` 格式（UTC）渲染在横条最底行

#### Scenario: 发布时间缺失
- **WHEN** 条目 PublishedAt 为 nil
- **THEN** 时间行显示处理时的当前 UTC 时间，格式同上

### Requirement: 半透明横条高度随内容动态调整
叠加横条 SHALL 根据实际文字行数动态调整高度，确保所有文字行均在横条范围内，同时保持底部对齐。

#### Scenario: 横条高度计算
- **WHEN** 标题共 N 行（1 ≤ N ≤ 3），加时间 1 行，共 N+1 行
- **THEN** 横条高度 = (N+1) × 14px + 6px 顶部内边距，横条底边与图片底边对齐

#### Scenario: 横条不透明度
- **WHEN** 叠加文字时
- **THEN** 黑色横条 alpha 值为 160（约 63% 不透明），宽度等于图片宽度
